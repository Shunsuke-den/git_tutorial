1. よく使うGitコマンド

まずはレポジトリの状態確認・差分確認など、日常的によく使うコマンドです。

現在の状態を確認する

変更されたファイルやコミットされていないファイルを確認します。

git status

差分を確認する

どの行が変更されたか、コミット前に確認します。

git diff

直前のコミットを修正する

コミットメッセージの修正やファイルの追加漏れに使います。

git commit --amend

2. コラボレーションのための操作

GitHubを使った開発でよく出てくるキーワードです。

リモートの変更を取り込む

GitHub上の最新の変更をローカルに取り込みます。

git pull

ブランチをマージする

別ブランチで行った作業を現在のブランチに統合します。

git merge

コンフリクト（競合）

同じ場所を複数人が編集したときに発生する「競合」のことです。

マージ時に手動で解決します。

Pull Request と GitHub Flow

GitHubでは、ブランチで作業 → Pull Request 作成 → レビュー → マージ、という流れをよく使います。

この一連の流れを「GitHub Flow」と呼びます。
